# StreamSuites-Members / FindMeHere v0.4.2-alpha

`FindMeHere v0.4.2-alpha` turns this repository into a real directory-first public surface instead of a leftover members shim. The release centers on canonical root-slug routing, clearer live and session tracking, and the branding and theme passes that now define the active `findmehere.live` experience.

## 📍 Directory Routing and Session Tracking

- Established `/` as the directory-first discovery surface and `/<slug>` as the canonical public profile route, with legacy compatibility paths preserved around the new SPA entry.
- Tightened same-origin profile hydration, canonical root-slug handling, and explicit page-visit tracking through the `directory` surface code used by runtime analytics and alerts.
- Wired FindMeHere traffic into the broader platform analytics model so visits and request-backed activity report separately from generic public traffic.

## 🎨 FindMeHere Redesign and Theme Behavior

- Reworked the FindMeHere presentation into the current premium dark-theme direction with dedicated branding, tighter hero copy, widened layout, and refined metallic/red visual treatment.
- Added a full light-theme counterpart plus persisted dark-light theme switching so the active surface no longer depends on a single presentation mode.
- Corrected theme-reactive title styling, footer anchoring, and icon-toggle behavior during the later stabilization passes.

## 🟢 Live Directory Presentation

- Continued the live-first presentation work with dedicated live views, live badges, pulsing avatar rings, and centralized live-status hydration.
- Kept the public profile experience share-first, with FindMeHere as the primary surface and StreamSuites profile links remaining secondary outbound links when provided by the authoritative payload.
